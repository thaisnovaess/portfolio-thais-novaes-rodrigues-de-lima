SM3 - Batalha de Modelos & Engenharia de Prompt (XML)

Este repositório documenta o terceiro experimento da disciplina, focado na análise comparativa de diferentes modelos de linguagem (LLMs) sob um protocolo de execução estrito. O objetivo central foi avaliar a fidelidade técnica e a conformidade das IAs ao processarem instruções estruturadas em XML para desenvolvimento Front-end.

🎯 Objetivos da Atividade Análise Crítica: Avaliar a precisão técnica de diferentes modelos em relação a instruções estritas.

Metacognição: Identificar as nuances, pontos fortes e limitações de cada arquitetura de IA para tarefas de desenvolvimento.

Fidelidade ao XML: Focar na obediência às tags e restrições, ignorando preferências estéticas subjetivas.

🛠️ A Tarefa: Single Page Application (SPA) Foi desenvolvido um Prompt Estruturado em XML para gerar uma página HTML única com CSS integrado. O teste consistiu em submeter o exato mesmo código para as seguintes ferramentas:

ChatGPT, Gemini, Claude, Qwen, DeepSeek, Grok e Maritaca.

Protocolo de Execução Submissão: Envio do prompt via dispositivo móvel.

Coleta de Dados: Verificação de tags ignoradas, respeito à paleta de cores e estrutura do código.

Métricas: Registro de consumo de tokens (estimado ou informado).

📝 Reflexão Crítica (Modelo Fink) Após a análise comparativa entre os modelos citados, os resultados foram consolidados com base na eficiência técnica:

Compreensão de Estrutura: Análise de qual modelo demonstrou maior "entendimento" da hierarquia das tags XML.

Verbosidade vs. Eficiência: Comparação do volume de tokens gerados para entregar o mesmo resultado funcional.

Tomada de Decisão:

Prototipagem Rápida: Escolha da ferramenta com melhor balanço entre velocidade e obediência inicial.

Códigos Complexos: Escolha da ferramenta com maior robustez e menor taxa de erro em instruções detalhadas.
